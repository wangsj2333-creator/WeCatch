# WeCatch V1 最终功能设计 & 执行计划

> 2026-03-27 brainstorm 会议产出。记录第一版最终功能范围和执行顺序。

---

## 三个实现任务

### 任务一：Popup 改造

**功能改动：**
- "打开 Dashboard" 按钮常驻（现在只在抓取完成后才显示）
- 文章列表过滤掉 0 条留言的文章

**UI：** 视觉重新设计（配色、字体、组件样式），样式在阶段一设计完再实现

---

### 任务二：Dashboard 改造

**布局：**
- 左右分栏
  - 左侧：文章列表（标题 + 总留言数），点击切换右侧内容
  - 右侧：当前文章的留言列表
  - 打开时默认选中第一篇文章

**右侧过滤栏：**
- 分类多选（标签 pill 形式，默认隐藏"无价值"类）
- 时间排序（最新优先 / 最早优先）

**留言展示：**
- 回复默认折叠，显示"N条回复"按钮，点击展开/收起

**导出：**
- 导出按钮放在左侧栏标题行右侧
- 点击弹出弹窗，列出所有文章并支持勾选（默认全选）
- 确认按钮实时显示"导出（N篇）"
- 导出内容：所选文章的全量留言（不受当前筛选影响）

**UI：** 视觉重新设计，样式在阶段一设计完再实现

---

### 任务三：LLM Prompt 优化（低优先级）

- 当前分类效果不理想，需重新设计 prompt
- 与其他任务完全无依赖，有空随时可做

---

## 阶段一：UI 设计规格

**完成标准：** 产出一份视觉规格，让实现者能直接照着写代码，不需要再做视觉决策。

**需要确定的内容：**
1. 整体风格基调（简洁工具感 / 轻量产品感 / 其他）
2. 配色方案（主色、背景色、文字色、分类标签色，具体 hex 值）
3. 组件样式规范（按钮、标签 pill、列表项的圆角、阴影、边框）
4. 字体与间距（字号层级、行高、基础间距单位）

**不需要：** 像素级完整设计稿

---

## 执行顺序

```
阶段一（前置）：UI 设计
  ↓ 确定 Popup + Dashboard 的视觉风格，为任务一、二提供输入
  ↓
阶段二：任务一 —— Popup 改造（功能 + UI）
  ↓ 改动量小，先完成
  ↓
阶段三：任务二 —— Dashboard 改造（功能 + UI）

阶段四：任务三 —— LLM Prompt 优化（独立，随时可插入）
```

**为什么这个顺序：**
- UI 设计是阶段二、三的前提，必须先做
- Popup 改动量小，先完成建立节奏
- Dashboard 是最大块，放在最后集中处理
- LLM Prompt 完全独立，不卡主线
